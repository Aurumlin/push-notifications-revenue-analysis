# ML Econ Project

Проект по дисциплине «Машинное обучение в экономике».

Цель проекта — смоделировать и проанализировать влияние подписки на push-уведомления на выручку пользователя в мобильном приложении. В работе используются синтетические данные, методы классификации, регрессии и оценки причинных эффектов.

## Описание проекта

В проекте рассматривается задача оценки влияния бинарной переменной воздействия:

```text
push_subscription = 1, если пользователь подписан на push-уведомления
push_subscription = 0, если пользователь не подписан на push-уведомления
```

Целевая переменная:

```text
revenue — выручка пользователя в мобильном приложении
```

Основная исследовательская гипотеза: подписка на push-уведомления может повышать выручку пользователя за счет роста вовлеченности, возврата в приложение и стимулирования покупок.

## Данные

Данные сгенерированы синтетически. Размер выборки — 10 000 наблюдений.

В датасете используются следующие признаки:

| Переменная          | Тип        | Описание                        |
| ------------------- | ---------- | ------------------------------- |
| `revenue`           | continuous | Выручка пользователя            |
| `push_subscription` | binary     | Подписка на push-уведомления    |
| `device_type`       | binary     | Тип устройства: iOS или Android |
| `age`               | continuous | Возраст пользователя            |
| `time_spent`        | continuous | Время, проведенное в приложении |
| `gender`            | binary     | Пол пользователя                |
| `new_user`          | binary     | Новый пользователь              |
| `added_to_cart`     | binary     | Добавление товара в корзину     |
| `loyalty`           | continuous | Лояльность пользователя         |

Переменная `loyalty` используется как ненаблюдаемая характеристика, влияющая одновременно на подписку на push-уведомления и на выручку. Это создает проблему эндогенности, которую далее учитывают методы оценки эффектов воздействия.

## Содержание работы

Проект состоит из нескольких частей:

1. Обоснование темы исследования
2. Генерация и предварительная обработка данных
3. Классификация подписки на push-уведомления
4. Регрессия для прогнозирования выручки
5. Оценка эффектов воздействия
6. Итоговый анализ результатов

## Используемые методы

### Классификация

Для прогнозирования подписки на push-уведомления сравниваются несколько моделей:

* K-Nearest Neighbors
* Random Forest
* Gradient Boosting
* Logistic Regression

Качество моделей оценивается с помощью:

* accuracy
* precision
* recall
* F1-score
* ROC-AUC
* кросс-валидации

Также проводится подбор гиперпараметров с помощью `RandomizedSearchCV`.

### Регрессия

Для прогнозирования выручки используются регрессионные модели машинного обучения. Качество оценивается по метрикам:

* RMSE
* MAPE

Проводится сравнение качества моделей до и после настройки гиперпараметров.

### Оценка эффектов воздействия

В проекте анализируется причинный эффект подписки на push-уведомления на выручку пользователя.

Рассматриваются:

* потенциальные исходы
* ATE
* LATE
* инструментальная переменная
* устойчивость результатов
* ограничения интерпретации

В качестве инструментальной переменной используется `device_type`, так как тип устройства влияет на вероятность подписки на push-уведомления через особенности операционных систем.

## Основные результаты

В задаче классификации наиболее стабильные результаты показала логистическая регрессия. Она продемонстрировала высокий уровень обобщающей способности и минимальный разрыв между качеством на обучающей и тестовой выборках.

Ансамблевые модели также показали хорошее качество, но были более склонны к переобучению по сравнению с логистической регрессией.

При анализе выручки было выявлено, что распределение целевой переменной смещено: большая часть пользователей достигает верхнего порога выручки. Это является важным ограничением при интерпретации результатов регрессионных моделей.
